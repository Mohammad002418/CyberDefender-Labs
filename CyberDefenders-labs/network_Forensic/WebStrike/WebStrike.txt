#Category:
Network Forensics.

#Tactics:
Initial Access , Execution , Persistence , Command and Control , Exfiltration.

#Tool:
Wireshark.

Scenario:
A suspicious file was identified on a company web server, raising alarms within the intranet. The Development team
flagged the anomaly, suspecting potential malicious activity. To address the issue, the network team captured 
critical network traffic and prepared a PCAP file for review.Your task is to analyze the provided PCAP file to
uncover how the file appeared and determine the extent of any unauthorized activity.


## Analysis Steps:
  

1- Open the PCAP in Wireshark – Start by loading the captured network traffic.

2- Filter HTTP traffic – Use http or tcp.port == 80 to locate web requests and responses.

3- Identify suspicious web shells – Look for unusual POST requests or encoded payloads in HTTP traffic.

4- Trace reverse shell connections – Filter by non-standard ports or tcp.stream to find outgoing connections from the server.

5- Check for data exfiltration – Look for large outbound transfers or repeated uploads to external IPs.

6- Document IOCs – Record IPs, domains, filenames, and commands observed during the compromise.

7- Summarize attack flow – Map the sequence: Initial Access → Execution → Persistence → C2 → Exfiltration.
