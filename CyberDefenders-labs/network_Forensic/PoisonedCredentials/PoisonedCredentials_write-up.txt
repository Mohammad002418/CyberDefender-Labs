# Lab Write-up: PoisonedCredentials

#Platform
CyberDefenders

#Category:
Network Forensics.

#Tactics:
Credential Access , Collection

## Scenario:
Your organization's security team has detected a surge in suspicious network activity. 
There are concerns that LLMNR (Link-Local Multicast Name Resolution) and NBT-NS (NetBIOS Name Service)
 poisoning attacks may be occurring within your network. These attacks are known for exploiting these protocols 
to intercept network traffic and potentially compromise user credentials. 
Your task is to investigate the network logs and examine captured network traffic.



## Tools Used:
Whireshark.


## Analysis Steps
 1- Open the provided PCAP file using Wireshark.

 2- Search for LLMNR and NBT-NS Traffic

   Apply the following display filter to locate suspicious name resolution traffic:

    llmnr || nbns

3- Identify Suspicious Responses

   Look for systems that respond to multiple LLMNR/NBT-NS queries.

   Attackers often respond pretending to be the requested host.

4-  Find the Rogue Machine

  Identify the IP address that sends many LLMNR/NBT-NS responses.

  This machine is likely the poisoning attacker.

  Determine Affected Systems

  Check the source IP addresses sending authentication requests.

  These are the victim machines affected by the poisoning attack.

5- Follow the Attack Timeline

   Observe the sequence:

   LLMNR/NBT-NS request

   Rogue response

   NTLM authentication attempt

6- Document Indicators of Compromise (IOCs)

  Rogue attacker IP
  
  Compromised usernames

  Victim machine IP addresses

  Suspicious LLMNR/NBT-NS responses

by taking advantage of benign network traffic from legitimate machines. Can you identify the specific mistyped query

of the second machine that received poisoned responses from the rogue machine?
Answer:192.168.232.176

Q4:We suspect that user accounts may have been compromised. To assess this, we must determine the username associated
 with the compromised account. What is the username of the account that the attacker compromised?
Answer:janesmith

Q5:As part of our investigation, we aim to understand the extent of the attacker's activities. 
What is the hostname of the machine that the attacker accessed via SMB?
Answer:AccountingPC.


#Lessons Learned:

LLMNR/NBT-NS poisoning can allow attackers to intercept authentication requests and capture NTLM credentials from victim machines.
Disabling LLMNR/NBT-NS and enforcing strong network monitoring can help prevent credential theft and detect rogue systems early.
