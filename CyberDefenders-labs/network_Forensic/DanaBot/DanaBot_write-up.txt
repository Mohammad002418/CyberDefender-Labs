##Lab write-up:DanaBot

#Category:
Network Forensics.

#Tactics: Execution , Command and Control.


#Tools:Wireshark , VirusTotal , ANY.RUN , Network Miner.


##Scenario:
The SOC team has detected suspicious activity in the network traffic, revealing that a machine has been compromised.
Sensitive company information has been stolen. Your task is to use Network Capture (PCAP) files
and Threat Intelligence to investigate the incident and determine how the breach occurred.


## Analysis Steps:
  

1- Open the PCAP in Wireshark – Load the captured network traffic for inspection.

2- Filter suspicious traffic – Use filters like tcp, http, or known malicious IPs to locate DanaBot communications.

3- Identify initial access patterns – Look for unusual downloads or scripts (e.g., JavaScript) delivered to the victim.

4- Deobfuscate malicious JavaScript – Examine downloaded scripts, decode or beautify them to understand their behavior.

5- Trace C2 communications – Identify outgoing connections to command-and-control servers using tcp.stream or ip.addr.

6- Extract IOCs – Record file hashes, IP addresses, domain names, and processes involved in the infection.

7- Verify artifacts – Cross-check files and IPs using VirusTotal or sandbox analysis on ANY.RUN.

8- Document attack flow – Map Execution → Command and Control → Data/Process manipulation for clarity.
